**1. Install dev dependencies:**
Run `cd packages/db && pnpm add -D @faker-js/faker tsx dotenv`

**2. Update packages/db/package.json:**
- Add `"db:seed": "npx prisma db seed"` to scripts
- Add prisma config to package.json:
```json
"prisma": {
  "seed": "npx tsx prisma/seed.ts"
}
```

**3. Create packages/db/src/seed-factories.ts:**
Export factory functions using @faker-js/faker. Use `faker.seed(12345)` at module level for reproducible data.

Factory functions (all return plain objects, not Prisma create inputs):

- `createUser(overrides?)`: generates clerkId (faker.string.uuid()), email, firstName, lastName, role (default CUSTOMER), avatar URL, phone
- `createCategory(depth, parentPath, parentId?, overrides?)`: generates name (faker.commerce.department()), slug, path (computed from parentPath), depth, position
- `createBrand(overrides?)`: generates name, slug, description, logo URL, website
- `createProduct(categoryId, brandId?, overrides?)`: generates name, slug (unique via faker.helpers.slugify + random suffix), description, price (Int, 999-99999), images (3 URLs), sku (alphanumeric 8 chars), productType, attributes JSON, status ACTIVE
- `createProductVariant(productId, overrides?)`: generates sku, price (Int), stock (0-100), isActive true
- `createReview(userId, productId, overrides?)`: generates rating (1-5), title, body, isVerifiedPurchase, status APPROVED
- `createCoupon(overrides?)`: generates code (uppercase 8 chars), discountType, discountValue, dates
- `createOrderData(userId, items, overrides?)`: generates orderNumber (ORD-XXXXXXXX), status, amounts (all Int cents), addresses, payment info
- `createCartData(overrides?)`: generates items with snapshots

All price values as integers (cents). Use faker.number.int() for price generation.

**4. Create packages/db/prisma/seed.ts:**

Main seed script that:

a) Imports PrismaClient, factory functions, Mongoose models, connectMongoDB, faker, dotenv
b) Calls dotenv.config() to load .env
c) Uses faker.seed(12345) for reproducibility
d) Cleans existing data in reverse-dependency order (delete all in both DBs first for idempotency)

Seed sequence:

**Users (5-10):**
- 1 SUPER_ADMIN user
- 2 ADMIN users
- 5-7 CUSTOMER users
- Each customer gets 1-2 addresses (one marked isDefault)

**Categories (10-15, 3 levels deep):**
- 4-5 root categories (Electronics, Clothing, Home & Garden, Sports, Books)
- 2-3 subcategories per root
- 1-2 sub-subcategories per subcategory
- Each leaf/mid category gets 2-3 CategoryAttributes

**Brands (5-8):**
- Various brands linked to products

**Tags (8-12):**
- Common e-commerce tags (new, sale, popular, limited edition, etc.)

**Collections (3-5):**
- Curated collections (Summer Sale, Best Sellers, New Arrivals, etc.)

**Option Groups + Values:**
- Size group: XS, S, M, L, XL, XXL
- Color group: Red, Blue, Green, Black, White, Navy, Gray
- Material group: Cotton, Polyester, Leather, Silk, Wool

**Products (~50 total, all types):**
- ~25 SIMPLE products across various categories
- ~10 VARIABLE products (each with 3-6 variants via OptionGroups)
- ~5 WEIGHTED products (with WeightedMeta)
- ~5 DIGITAL products (with DigitalMeta)
- ~5 BUNDLED products (with BundleItems referencing other products)
- Each product assigned to a category, optionally to a brand
- Products linked to tags (2-4 tags each) and collections (some products in collections)
- Variable products: create ProductVariants connected to OptionValues via VariantOption

**Reviews (20-30):**
- Spread across products, from customer users
- Mix of ratings (1-5), some with photos, some verified purchases

**Coupons (3-5):**
- WELCOME10: 10% off, no minimum
- SAVE20: $20 off, $100 minimum
- FREESHIP: free shipping, $50 minimum
- FLASH25: 25% off, limited usage, specific date range

**Promotions (2-3):**
- Buy 3+ get 10% off (TIERED_PRICING)
- Buy 2 get 1 free (BOGO)

**Shipping Zones + Methods:**
- US Domestic zone (flat rate $5.99, free above $75)
- International zone (weight-based $15.99 base)
- Express method available for US zone ($14.99 flat)

**Warehouses (2-3):**
- East Coast warehouse, West Coast warehouse, Central warehouse
- Each with address, priority

**Inventory Items:**
- For each ProductVariant, create 1-2 InventoryItems across warehouses
- Random quantities (5-100)
- Some with low stock (below threshold) for testing alerts

**MongoDB Seeds:**
Connect to MongoDB (use MONGODB_URI from env, fallback to mongodb://localhost:27017/ecommerce).

- 10-15 Orders across various statuses (pending, paid, processing, shipped, delivered, cancelled)
- Each order has 1-4 line items with denormalized product data from seeded products
- Orders have statusHistory entries
- 3-5 Carts (2 guest with sessionId + expiresAt set to 7 days from now, 2-3 authenticated with userId)
- Cart items have price/name snapshots

Script prints progress messages: "Seeding users... Created X users", etc.
Script wraps in try/catch/finally with prisma.$disconnect() and mongoose.disconnect().

Use prisma.MODEL.deleteMany() for cleanup (in correct order to avoid FK violations).
Use prisma.MODEL.create() or prisma.MODEL.createMany() for seeding.
