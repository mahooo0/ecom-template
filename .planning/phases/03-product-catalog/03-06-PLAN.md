   - Export `csvUpload` middleware:
     ```typescript
     export const csvUpload = multer({
       storage: multer.memoryStorage(),
       limits: { fileSize: 10 * 1024 * 1024 }, // 10MB max
       fileFilter: (req, file, cb) => {
         const allowedMimes = ['text/csv', 'application/vnd.ms-excel'];
         if (allowedMimes.includes(file.mimetype) || file.originalname.endsWith('.csv')) {
           cb(null, true);
         } else {
           cb(new Error('Only CSV files are allowed'));
         }
       },
     }).single('file');
     ```

1. Add `importFromCsv` method to `apps/server/src/modules/product/product.service.ts`:
   - Import Papa from 'papaparse'
   - Method signature: `async importFromCsv(fileBuffer: Buffer): Promise<ImportResult>`
   - ImportResult type: `{ total: number; imported: number; failed: number; errors: Array<{ row: number; field?: string; message: string }> }`

   - Implementation:
     - Convert buffer to string: `const csvString = fileBuffer.toString('utf-8')`
     - Parse with Papa.parse(csvString, { header: true, skipEmptyLines: true, transformHeader: (h) => h.trim().toLowerCase() })
     - Expected CSV columns: name, description, price, sku, productType, categoryId, brandId (optional), status, images (pipe-separated URLs)
     - Type-specific columns:
       - WEIGHTED: weightUnit, pricePerUnit, minWeight, maxWeight, stepWeight
       - DIGITAL: fileUrl, fileName, fileSize, fileFormat, maxDownloads, accessDuration
       - BUNDLED: bundleProductIds (pipe-separated), bundleQuantities (pipe-separated), bundleDiscounts (pipe-separated)
       - VARIABLE: not supported in CSV import (too complex -- variants must be created via form)

     - For each row, validate:
       - Parse price to integer (handle "$12.99" -> 1299 and "1299" as cents)
       - Map productType to enum
       - Build ProductFormData object from CSV columns
       - Validate with productSchema.safeParse()
       - If valid: call this.create(validatedData) wrapped in try/catch
       - If invalid: add to errors array with row number and Zod error details

     - Process rows sequentially (not parallel) to avoid overwhelming the database
     - Wrap each create in try/catch to capture DB errors (e.g., duplicate SKU)
     - Return ImportResult with counts and error details
