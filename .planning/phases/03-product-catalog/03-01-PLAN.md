Output: Zod validation schemas (shared), expanded product service with type-aware create/update, slug utility, and new API endpoints for status management and admin listing.

2. Rewrite `apps/server/src/modules/product/product.service.ts` to handle all 5 product types:
   - `create(data: ProductFormData)`:
     - Generate unique slug via generateUniqueSlug(data.name)
     - Use `prisma.product.create()` with type-conditional nested creates:
       - For VARIABLE: `{ variants: { create: data.variants.map(v => ({ sku: v.sku, price: v.price, stock: v.stock, isActive: v.isActive, images: v.images || [], options: { create: v.options.map(o => ({ optionId: o.valueId })) } })) } }`
       - For WEIGHTED: `{ weightedMeta: { create: data.weightedMeta } }`
       - For DIGITAL: `{ digitalMeta: { create: data.digitalMeta } }`
       - For BUNDLED: `{ bundleItems: { create: data.bundleItems.map(bi => ({ productId: bi.productId, quantity: bi.quantity, discount: bi.discount })) } }`
     - Handle tagIds: if provided, create ProductTag records
     - Handle collectionIds: if provided, create ProductCollection records
     - Include all relations in return
     - Emit 'product.created' event

   - `update(id: string, data)`:
     - If name changed, regenerate slug with generateUniqueSlug(data.name, id)
     - Use `prisma.product.update()` with conditional nested updates
     - For VARIABLE type: delete existing variants, recreate (simplest approach for complex nested updates)
     - For WEIGHTED/DIGITAL: upsert metadata
     - For BUNDLED: delete existing bundle items, recreate
     - Handle tagIds/collectionIds: delete existing, recreate
     - Emit 'product.updated' event

   - `updateStatus(id: string, status: ProductStatus)`:
     - Update status field
     - If status is ARCHIVED, set isActive to false
     - If status is ACTIVE, set isActive to true
     - Emit 'product.updated' event

   - `getAll(options: { page, limit, status?, productType?, search?, sortBy?, sortOrder? })`:
     - Build where clause with optional status, productType, and search (name contains, case insensitive)
     - Offset pagination with skip/take
     - Dynamic orderBy based on sortBy (createdAt, name, price, updatedAt) and sortOrder (asc/desc)
     - Include category, brand, _count of variants
     - Return PaginatedResponse format

   - `getById(id: string)`:
     - Include ALL relations: category, brand, variants with options (with option group and value), digitalMeta, weightedMeta, bundleItems with product, tags with tag, collections with collection, reviews (count + average)
     - Throw 404 if not found

   - `getBySlug(slug: string)`:
     - Same as getById but lookup by slug
     - Add where clause: `{ status: 'ACTIVE', isActive: true }` for client-facing

   - `delete(id: string)`:
     - Delete product (cascading deletes handle relations per schema)
     - Emit 'product.deleted' event

   - `bulkUpdateStatus(ids: string[], status: ProductStatus)`:
     - Use prisma.product.updateMany({ where: { id: { in: ids } }, data: { status, isActive: status === 'ACTIVE' } })
     - Emit events for each id

   - `bulkDelete(ids: string[])`:
     - Use prisma.product.deleteMany({ where: { id: { in: ids } } })
     - Emit events for each id
