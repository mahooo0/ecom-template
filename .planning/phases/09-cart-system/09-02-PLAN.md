     - `couponCode: string | null`
     - `discountAmount: number` (cents)
     - `applyCoupon: (code: string, discount: number) => void`
     - `removeCoupon: () => void`
     - `subtotal: () => number`
     - `setItems: (items: CartItem[]) => void` (for server cart hydration)
   - Add initial values: couponCode: null, discountAmount: 0
   - Implement applyCoupon: set({ couponCode: code, discountAmount: discount })
   - Implement removeCoupon: set({ couponCode: null, discountAmount: 0 })
   - Implement subtotal: return get().items.reduce((sum, item) => sum + item.price * item.quantity, 0)
   - Implement setItems: set({ items })
   - Keep all existing methods (addItem, removeItem, updateQuantity, clearCart, totalItems) unchanged

2. Create `apps/client/src/hooks/use-cart-sync.ts`:
   - Mirror `use-wishlist-sync.ts` exactly in structure
   - 'use client' directive at top
   - Import useEffect, useRef from 'react'
   - Import useUser, useAuth from '@clerk/nextjs'
   - Import useCartStore from '@/stores/cart-store'
   - Export function useCartSync()
   - Use syncedRef = useRef(false) to prevent re-sync
   - useEffect depends only on [isSignedIn]
   - On isSignedIn becoming true:
     a. Set syncedRef.current = true
     b. Get token via getToken()
     c. If items.length > 0: AWAIT (not fire-and-forget, per research Pitfall 3) POST to /api/cart/merge with { items, couponCode } from store. Use Authorization: `Bearer ${token}` header.
     d. After merge completes: GET /api/cart to fetch authoritative server cart
     e. If response ok: parse JSON, call setItems(data.data.items), and update couponCode/discountAmount from server data
     f. Wrap everything in try/catch, silently fail (local state remains intact)
   - IMPORTANT: Per research Pitfall 3, merge must complete BEFORE fetching server cart (sequential, not parallel). This is the one difference from wishlist sync.
