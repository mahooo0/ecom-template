  <action>
**HelpTooltip.tsx** (identical in both apps, `'use client'`):
- Props: `{ content: string; side?: 'top' | 'bottom' | 'left' | 'right' }` (default side: 'top')
- State: `visible` boolean
- Renders a relative inline-block wrapper containing:
  - A button (type="button") with `?` text, styled: `w-5 h-5 rounded-full bg-gray-200 text-gray-600 text-xs flex items-center justify-center hover:bg-gray-300 cursor-help`
  - onMouseEnter sets visible true, onMouseLeave sets visible false, onClick toggles
  - When visible, renders a positioned tooltip div:
    - z-50, absolute, w-52, bg-gray-900 text-white text-xs rounded-md p-2.5 shadow-lg
    - Position based on `side` prop (top: bottom-7, bottom: top-7, left: right-7, right: left-7)
    - Arrow triangle using CSS border trick pointing toward the button
  - Content rendered as text inside the tooltip
- Export as named `HelpTooltip`

**useTour.ts** (in both apps, `'use client'` compatible):
- Exports `useTour(tourId: string, steps: DriveStep[])` hook
- Returns `{ startTour: () => void, isCompleted: boolean, resetTour: () => void }`
- On `startTour()`:
  - Checks `localStorage.getItem('tour_completed_' + tourId)` -- if 'true', returns early (no-op)
  - Creates Driver.js instance with `driver({ showProgress: true, allowClose: true, showButtons: ['previous', 'next', 'close'], doneBtnText: 'Got it!', steps, onDestroyed: () => localStorage.setItem('tour_completed_' + tourId, 'true') })`
  - Calls `driverObj.drive()`
- `isCompleted` reads from localStorage
- `resetTour()` removes the localStorage key
- Import `driver` from 'driver.js' and `import 'driver.js/dist/driver.css'`
- Import type `DriveStep` from 'driver.js'

Create both files identically in `apps/client/src/components/guidance/` and `apps/admin/src/components/guidance/`.
