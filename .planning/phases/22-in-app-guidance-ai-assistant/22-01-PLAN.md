  <name>Task 1: Create client AI chat API route with rate limiting</name>
  <files>apps/client/src/app/api/chat/route.ts</files>
  <action>
Create `apps/client/src/app/api/chat/route.ts` with:

1. **Rate limiter** at module level: IP-based Map with 10 requests per 60-second window. `checkRateLimit(ip: string): boolean` function that tracks count and resetAt per IP. Return 429 Response if limit exceeded.

2. **POST handler** that:
   - Extracts IP from `x-forwarded-for` header (fallback to 'unknown')
   - Checks rate limit, returns 429 if blocked
   - Parses `{ messages, context }` from request body (context is optional: `{ page?: string, productName?: string, category?: string }`)
   - Builds a dynamic system prompt:
     ```
     You are a helpful shopping assistant for an e-commerce store.
     Store: E-Commerce Store.
     Current page: {context.page ?? 'homepage'}.
     {If productName: User is viewing: "{productName}" in {category}.}
     Policies: Free shipping over $50. 30-day returns. We accept all major credit cards.
     Help users find products, answer questions about orders, shipping, and returns.
     Keep responses concise (2-3 sentences max).
     If asked about specific inventory or real-time pricing, suggest contacting support for accurate data.
     ```
   - Calls `streamText` with `openai('gpt-4o-mini')`, the system prompt, `convertToModelMessages(messages)`, and `maxTokens: 500`
   - Returns `result.toUIMessageStreamResponse()`

3. **Export** `const maxDuration = 30;` at top level for Vercel serverless timeout.

Use imports: `streamText`, `UIMessage`, `convertToModelMessages` from 'ai'; `openai` from '@ai-sdk/openai'.

Add `OPENAI_API_KEY` to `.env.example` if it does not exist (do NOT create .env, only .env.example).
  </action>

- Renders a form with text input (placeholder: "Ask me anything...") and a send button
- On submit, calls `onSend(inputValue)` and clears input
- Disable input and button when `isLoading` is true
- Tailwind styling: border-t, p-3, flex gap-2, rounded input, blue send button

**ChatMessages.tsx** (`'use client'`):
- Props: `{ messages: Array<{ id: string; role: string; parts: Array<{ type: string; text?: string }> }>, isStreaming: boolean }`
- Maps over messages, rendering each message's text parts
- User messages aligned right with blue background, assistant messages aligned left with gray background
- Shows "Thinking..." text when `isStreaming` is true and last message is from user
- Auto-scrolls to bottom on new messages using a ref + useEffect
- Renders a welcome message when messages array is empty: "Hi! I'm your shopping assistant. How can I help you today?"
