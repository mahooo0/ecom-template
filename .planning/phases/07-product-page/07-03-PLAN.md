  <action>
    Create `'use client'` component `VariantSelector`.

    Props:
    ```typescript
    interface VariantSelectorProps {
      variants: ProductVariantDetail[];
      onVariantChange: (variant: ProductVariantDetail | null) => void;
    }
    ```

    **Build variant combination matrix:**
    - `buildVariantMatrix(variants)`: Create a Map from sorted option key string (e.g., "Color:Red|Size:M") to variant data { variantId, stock, price, images, sku }.
    - `extractOptionGroups(variants)`: Derive unique option groups and their values. Each group: { name, displayName, values: string[] }.

    **Rendering:**
    - For each option group, render a `<select>` dropdown with `<option>` elements.
    - Label each dropdown with `group.displayName` (e.g., "Size", "Color").
    - Default selection: initialize with first variant's options (so page loads with a valid selection).

    **Availability logic:**
    - When user changes one dropdown, determine which values in OTHER dropdowns are still available.
    - For each value in each group, check if any variant exists with that value + all other current selections. If no valid combination exists, mark that `<option>` as `disabled`.
    - Disabled options show grayed out text.

    **On selection change:**
    - Build the selection key from all current dropdown values.
    - Look up the variant in the matrix.
    - Call `onVariantChange(matchedVariant)` -- pass null if no valid combination (shouldn't happen if defaults are valid).

    **State:** `selections: Record<string, string>` -- maps group name to selected value.

    Use Tailwind for styling. Dropdowns should have consistent appearance: `border border-gray-300 rounded-lg px-3 py-2 bg-white` with `disabled:opacity-50 disabled:cursor-not-allowed` on options.
  </action>
