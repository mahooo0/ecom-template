    **First, update getBySlug in product.service.ts** to also include `category: { include: { attributes: true } }` so the specs table has CategoryAttribute display names. Change the existing `category: true` to `category: { include: { attributes: true } }` in the getBySlug include block.

    **Create `product-page-client.tsx` as a separate `'use client'` component file.** This is a client island that manages all interactive state for the product page above-the-fold area. It receives the full product data as props from the Server Component and manages:
    - `selectedVariant` state (initialized from first variant for VARIABLE products, null otherwise)
    - `currentImages` derived from selectedVariant?.images (if non-empty) or product.images
    - `currentPrice` derived from selectedVariant?.price or product.price
    - `currentStock` derived from selectedVariant?.stock or product.variants[0]?.stock or 0
    - `weightTotal` state for WEIGHTED products (passed to WeightedQuantitySelector)
    - Variant change handler that updates selectedVariant and all derived values

    ProductPageClient renders the above-the-fold two-column layout:
    - **Left column:** `<ProductImageGallery images={currentImages} productName={product.name} />`
    - **Right column (product info + purchase area):**
      - Breadcrumb (category path if available -- simple text breadcrumb)
      - Product name (`<h1>`)
      - Brand name (if product.brand)
      - Rating display (StarRating + review count)
      - Price display (formatted from cents, show compareAtPrice strikethrough if on sale)
      - **VARIABLE type:** `<VariantSelector>` -- variant changes update images, price, stock, SKU
      - **WEIGHTED type:** `<WeightedQuantitySelector>` with product.weightedMeta data
      - **All types:** `<StockStatus stock={currentStock} />`
      - **All types:** `<AddToCartButton>` with current product/variant data
      - SKU display (small text)
      - Tags (if any)
    - Layout: `grid grid-cols-1 lg:grid-cols-2 gap-8`

    **Create `[slug]/page.tsx` as an async Server Component:**

    ```typescript
    import { notFound } from 'next/navigation';
    import { api } from '@/lib/api';
    import type { ProductDetail } from '@/types/product-detail';
    import { ProductPageClient } from './product-page-client';
    ```

    Data fetching:
    - `const { slug } = await params;`
    - Fetch product via `api.products.getBySlug(slug)`. If fails or returns null, call `notFound()`.
    - Cast result to `ProductDetail` type.
    - Fetch related and FBT in parallel with `Promise.allSettled([api.products.getRelated(product.id), api.products.getFrequentlyBoughtTogether(product.id)])`. Extract data from fulfilled results, default to empty arrays.

    **Page layout in page.tsx:**

    Above the fold:
    - `<ProductPageClient product={product} />` -- the client island handles all interactive above-fold content

    Below the fold (full width sections, stacked vertically, rendered by the Server Component):
    - **DIGITAL type:** `<DigitalProductInfo>` with product.digitalMeta
    - **BUNDLED type:** `<BundleItemsList>` with product.bundleItems and product.price
    - **All types (if attributes exist):** `<ProductSpecsTable>` with product.attributes and category.attributes
    - **All types:** `<ReviewsPlaceholder>` with averageRating and reviewCount
    - **If relatedProducts.length > 0:** `<RelatedProductsCarousel products={relatedProducts} />`
    - **If fbtProducts.length > 0:** `<FrequentlyBoughtTogether products={fbtProducts} currentProduct={...} />`

    **Page metadata:**
    - Export `generateMetadata` function that fetches product and returns `{ title: product.name, description: product.description?.slice(0, 160) }` for SEO.
