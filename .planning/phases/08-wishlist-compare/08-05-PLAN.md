- Import eventBus from '../../common/events/event-bus.js'
- Import prisma from '@repo/db'

Export function registerWishlistEventListeners():

**Price drop listener** -- eventBus.on('product.updated'):
- Receives { productId }
- Fetch the product's current price: prisma.product.findUnique({ where: { id: productId }, select: { price: true } })
- If product not found, return early
- Find all wishlist items for this product where notifyOnPriceDrop=true AND priceAtAdd > product.price (price has decreased):
  ```
  prisma.wishlistItem.findMany({
    where: { productId, notifyOnPriceDrop: true, priceAtAdd: { gt: product.price } },
    include: { wishlist: { select: { userId: true } } }
  })
  ```
- If any items found, extract unique userIds and emit:
  ```
  eventBus.emit('wishlist.priceDrop', {
    productId,
    oldPrice: items[0].priceAtAdd, // representative -- actual priceAtAdd varies per user
    newPrice: product.price,
    affectedUserIds: [...new Set(items.map(i => i.wishlist.userId))]
  });
  ```
- After emitting, update those items' priceAtAdd to the new price so the same drop doesn't trigger again:
  ```
  prisma.wishlistItem.updateMany({
    where: { id: { in: items.map(i => i.id) } },
    data: { priceAtAdd: product.price }
  })
  ```
- Wrap everything in try/catch -- log errors but don't crash (event listeners must be resilient)

**Restock listener** -- eventBus.on('inventory.stockUpdated'):
- Receives { variantId, warehouseId, quantity, available }
- Only proceed if available > 0 (item is back in stock)
- Find the product associated with this variant: prisma.productVariant.findUnique({ where: { id: variantId }, select: { productId: true } })
- If no variant found, return early
- Check if this product was previously out of stock (all variants had 0 available). Query: prisma.inventoryItem.aggregate({ where: { variantId: { in: variantIdsForProduct }, NOT: { variantId } }, _sum: { quantity: true } }). Actually simpler: since we just got a restock event with available > 0, and we want to know if it was OUT of stock before this event, we can check if the previous available was 0 (available - quantity change). However, the simplest approach: just check if any other inventory items for the same product have available > 0. If this is the ONLY item with stock, it's a restock event.
- Simpler approach: Just emit for any stock increase on a wishlisted product. Phase 17 can deduplicate. Find wishlist items where notifyOnRestock=true for this productId:
  ```
  prisma.wishlistItem.findMany({
    where: { productId: variant.productId, notifyOnRestock: true },
    include: { wishlist: { select: { userId: true } } }
  })
  ```
- If items found, emit 'wishlist.restock' with affectedUserIds
- Wrap in try/catch for resilience
