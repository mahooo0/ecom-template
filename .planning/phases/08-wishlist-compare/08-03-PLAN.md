  <action>
1. Create apps/client/src/hooks/use-wishlist-sync.ts:
- 'use client' hook
- Uses useUser() from @clerk/nextjs to detect isSignedIn
- Uses useWishlistStore items
- useRef(false) as syncedRef to prevent double-sync
- useEffect: if isSignedIn AND !syncedRef.current AND items.length > 0:
  - Set syncedRef.current = true
  - POST to /api/wishlist/sync with { items } (include Authorization header from useAuth getToken)
  - Fire-and-forget (catch errors silently — local state remains intact)
  - On success, optionally fetch full wishlist from GET /api/wishlist to update store with any server-side items the user had from another device
- This hook should be placed in the root layout client wrapper so it runs on every page load

2. Create apps/client/src/components/wishlist/price-drop-badge.tsx as 'use client' component:
- Props: { priceAtAdd: number; currentPrice: number }
- If currentPrice >= priceAtAdd, return null (no badge)
- Otherwise, show a small green badge: "Price dropped! Save $X.XX" where X = (priceAtAdd - currentPrice) / 100
- Tailwind styling: bg-green-100 text-green-800 text-xs font-medium px-2 py-0.5 rounded-full

3. Create apps/client/src/app/wishlist/page.tsx as Server Component:
- Export metadata: { title: 'My Wishlist' }
- Renders WishlistPageClient

4. Create apps/client/src/app/wishlist/wishlist-page-client.tsx as 'use client':
- For authenticated users: fetch wishlist items from GET /api/wishlist on mount (with Clerk token)
- For guest users: read from useWishlistStore and fetch product details for each productId (GET /api/products/:id or similar)
- Display as responsive grid of product cards (reuse ProductCard pattern) with:
  - WishlistButton (heart already filled since item is in wishlist)
  - PriceDropBadge for each item (compare priceAtAdd from store/DB with current product price)
  - "Add to Cart" button on each card
  - "Remove" button (or rely on heart toggle)
- Per-item notify toggles: two small toggle switches for "Price drop alerts" and "Restock alerts" (only shown for authenticated users). These call PATCH /api/wishlist/:productId/notify.
- Empty state: heart icon illustration + "Your wishlist is empty" message + "Browse Products" link to /products
- Handle hydration: show skeleton/loading until mounted
