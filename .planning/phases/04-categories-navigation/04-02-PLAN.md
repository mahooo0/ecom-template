Create three server modules following the established controller/service/routes pattern. Each module is simple CRUD -- no complex business logic.

**Collection module:**

`collection.schemas.ts`:
- `createCollectionSchema`: body { name (string, 1-100), description? (string), image? (string), isActive? (boolean, default true) }
- `updateCollectionSchema`: body { name?, description?, image?, isActive?, slug? (regex /^[a-z0-9-]+$/) }
- `addProductSchema`: body { productId (string), position? (number, default 0) }

`collection.service.ts` (CollectionService singleton):
- `getAll(page=1, limit=20)`: Paginated list with product count. Return { data, total, page, limit, totalPages }.
- `getById(id)`: Find with products included (include: { products: { include: { product: true }, orderBy: { position: 'asc' } } }). AppError(404) if not found.
- `getBySlug(slug)`: Find by slug with products. AppError(404) if not found.
- `create(data)`: Generate slug from name (lowercase, replace spaces with hyphens, strip special chars). Create collection.
- `update(id, data)`: Update collection fields. If slug not provided and name changed, regenerate slug.
- `delete(id)`: Delete collection (cascades to ProductCollection via schema).
- `addProduct(collectionId, productId, position)`: Create ProductCollection entry. Handle unique constraint error gracefully (AppError(400, 'Product already in collection')).
- `removeProduct(collectionId, productId)`: Delete ProductCollection entry.
- `reorderProducts(collectionId, orderedProductIds)`: Update position for each product in collection.

`collection.controller.ts` (singleton): Standard controller methods (getAll, getById, getBySlug, create, update, delete, addProduct, removeProduct, reorderProducts).

`collection.routes.ts`:
- Public: `GET /` (list), `GET /slug/:slug`, `GET /:id`
- Admin: `POST /` (create), `PUT /:id`, `DELETE /:id`, `POST /:id/products` (add product), `DELETE /:id/products/:productId` (remove product), `PATCH /:id/reorder`

**Brand module:**

`brand.schemas.ts`:
- `createBrandSchema`: body { name (string, 1-100), description? (string), logo? (string), website? (string URL) }
- `updateBrandSchema`: body { name?, description?, logo?, website?, slug? }

`brand.service.ts` (BrandService singleton):
- `getAll(page=1, limit=20)`: Paginated list with product count.
- `getById(id)`: Find by id. AppError(404) if not found.
- `getBySlug(slug)`: Find by slug. AppError(404) if not found.
- `create(data)`: Generate slug from name. Create brand.
- `update(id, data)`: Update brand.
- `delete(id)`: Delete brand (products set brandId to null via onDelete: SetNull).

`brand.controller.ts` + `brand.routes.ts`:
- Public: `GET /` (list), `GET /slug/:slug`, `GET /:id`
- Admin: `POST /`, `PUT /:id`, `DELETE /:id`

**Tag module:**

`tag.schemas.ts`:
- `createTagSchema`: body { name (string, 1-50) }

`tag.service.ts` (TagService singleton):
- `getAll(page=1, limit=50)`: Paginated list with product count.
- `create(data)`: Generate slug from name. Create tag.
- `delete(id)`: Delete tag (cascades to ProductTag via schema).

`tag.controller.ts` + `tag.routes.ts`:
- Public: `GET /` (list)
- Admin: `POST /`, `DELETE /:id`
