**apps/admin/src/app/dashboard/categories/category-form.tsx ('use client'):**

Create a form component for creating and editing categories. Use react-hook-form with zodResolver (already available in project from Phase 3).

Props: `{ category?: Category; categories: Category[]; onSuccess: () => void }`

Form fields:
1. **Name** - text input, required, 1-100 chars
2. **Description** - textarea, optional
3. **Parent Category** - select dropdown populated from `categories` prop (exclude self and descendants for edit mode). Show "None (Root Category)" as first option.
4. **Image** - text input for URL (simple for now, image upload can reuse Cloudinary from Phase 3 later)
5. **SEO Section** (collapsible or separate section):
   - **Meta Title** - text input, max 60 chars, with character counter
   - **Meta Description** - textarea, max 160 chars, with character counter
   - **Slug** - text input (auto-generated from name, editable), regex validation /^[a-z0-9-]+$/, show preview URL: `/categories/{slug}`

Zod schema for client-side validation (mirror server schema):
```typescript
const categoryFormSchema = z.object({
  name: z.string().min(1).max(100),
  description: z.string().optional(),
  parentId: z.string().nullable().optional(),
  image: z.string().url().optional().or(z.literal('')),
  metaTitle: z.string().max(60).optional(),
  metaDescription: z.string().max(160).optional(),
  slug: z.string().regex(/^[a-z0-9-]+$/).optional(),
});
```

On submit:
- If editing (category prop exists): call api.categories.update(category.id, formData, token)
- If creating: call api.categories.create(formData, token)
- On success: call onSuccess() callback (which does router.refresh())
- On error: display error message from API

Style with Tailwind: card layout, labeled form fields, save and cancel buttons.

Auto-generate slug from name on create mode (debounced, only if slug field hasn't been manually edited).

**apps/admin/src/app/dashboard/categories/attribute-manager.tsx ('use client'):**

Props: `{ categoryId: string; attributes: CategoryAttribute[] }`

Component shows:
1. List of existing attributes as a table with columns: Name, Key, Type, Values, Filterable, Actions (edit/delete)
2. "Add Attribute" button that shows an inline form

Attribute form fields:
- **Name** - text input (display name, e.g., "Screen Size")
- **Key** - text input (machine key, e.g., "screen_size"), auto-generated from name, regex /^[a-z_][a-z0-9_]*$/
- **Type** - select: SELECT, RANGE, BOOLEAN, TEXT
- **Values** - tag input or textarea (one per line) - only shown when type is SELECT. Parse textarea into string[] on submit.
- **Unit** - text input (e.g., "inch", "GB") - shown for SELECT and RANGE types
- **Filterable** - checkbox, default true
- **Required** - checkbox, default false

On submit:
- Call api.categories.createAttribute(categoryId, data, token) for new
- Call api.categories.updateAttribute(attributeId, data, token) for edit
- Refresh attribute list

On delete: confirm dialog, then api.categories.deleteAttribute(attributeId, token)

Style: clean table layout, inline form below table, Tailwind classes consistent with admin app.
