**category.schemas.ts:** Create Zod validation schemas following the product.schemas.ts pattern:

1. `createCategorySchema` - body: { name (string, 1-100 chars), description? (string), image? (string URL), parentId? (string), metaTitle? (string, max 60), metaDescription? (string, max 160), position? (number, default 0) }. Slug auto-generated server-side.

2. `updateCategorySchema` - body: same fields as create but all optional, plus slug? (string, regex /^[a-z0-9-]+$/), metaTitle?, metaDescription?

3. `moveCategorySchema` - body: { newParentId (string | null), position (number >= 0) }

4. `createCategoryAttributeSchema` - body: { name (string, 1-50), key (string, regex /^[a-z_][a-z0-9_]*$/), type (enum: SELECT, RANGE, BOOLEAN, TEXT), values? (string[]), unit? (string), isFilterable? (boolean, default true), isRequired? (boolean, default false), position? (number, default 0) }. Add refinement: if type is SELECT, values must be non-empty array.

5. `updateCategoryAttributeSchema` - body: all fields optional, same refinement for SELECT type.

**category.service.ts:** Create CategoryService class following the singleton pattern (export const categoryService = new CategoryService()):

Methods:
- `getAll()`: Return all categories ordered by [depth asc, position asc]. Include children count.
- `getById(id)`: Find by id, include attributes and children. Throw AppError(404) if not found.
- `getBySlug(slug)`: Find by slug, include attributes. Throw AppError(404) if not found.
- `getTree()`: Find all categories, build and return nested tree structure (root nodes with recursive children). Order by position within each level.
- `create(data)`: Generate slug from name using slugify (lowercase, strip special chars, append counter if duplicate). Calculate path as parentPath + '/' + slug (or '/' + slug for root). Set depth from parent.depth + 1 (or 0 for root). Validate max depth of 5 (throw AppError(400) if parent.depth >= 5). Create and return.
- `update(id, data)`: Update category fields. If name changed and slug was auto-generated (matches slugify of old name), regenerate slug and update path accordingly. If slug explicitly provided, use it and update path. If path changes, update all descendant paths in a transaction.
- `delete(id)`: Check for child categories (throw AppError(400, 'Cannot delete category with children')). Check for products (throw AppError(400, 'Cannot delete category with products')). Delete category.
- `move(id, newParentId, position)`: Implement from research Pattern 2. Check circular reference (target is not descendant of source). Use $transaction: update category's parentId, path, depth, position; update all descendants' path and depth. Throw AppError(400) for circular reference.
- `reorderSiblings(parentId, orderedIds)`: Update position for each id in array.
- `getDescendantIds(id)`: Get all descendant category IDs using path startsWith.
- `getAncestors(categoryId)`: Parse materialized path, fetch all ancestor categories ordered by depth.

Attribute methods on same service:
- `getAttributes(categoryId)`: Get all attributes for a category, ordered by position.
- `createAttribute(categoryId, data)`: Create attribute, validate unique key per category.
- `updateAttribute(id, data)`: Update attribute fields.
- `deleteAttribute(id)`: Delete attribute.

Helper functions (private or module-level):
- `generateUniqueSlug(name)`: Slugify name, check uniqueness against prisma.category, append -2, -3 if needed.
- `buildTreeFromFlat(categories)`: Convert flat category array to nested tree.
- `getAncestorPathsFromPath(path)`: Split materialized path into ancestor paths array.

Use `import { prisma } from '@repo/db';` for database access.
Use `import { AppError } from '../../common/middleware/error-handler.js';` for errors.
Use `.js` extensions on all relative imports (ESM convention).

**category.controller.ts:** Create CategoryController class following the product.controller.ts pattern:

Methods (all async with try/catch + next(error)):
- `getAll(req, res, next)`: Call categoryService.getAll(), return { success: true, data }
- `getTree(req, res, next)`: Call categoryService.getTree(), return { success: true, data }
- `getById(req, res, next)`: Call categoryService.getById(req.params.id), return { success: true, data }
- `getBySlug(req, res, next)`: Call categoryService.getBySlug(req.params.slug), return { success: true, data }
- `create(req, res, next)`: Call categoryService.create(req.body), return 201 with { success: true, data }
- `update(req, res, next)`: Call categoryService.update(req.params.id, req.body), return { success: true, data }
- `delete(req, res, next)`: Call categoryService.delete(req.params.id), return { success: true, message: 'Category deleted' }
- `move(req, res, next)`: Call categoryService.move(req.params.id, req.body.newParentId, req.body.position), return { success: true, data }
- `reorder(req, res, next)`: Call categoryService.reorderSiblings(req.body.parentId, req.body.orderedIds), return { success: true }
- `getAttributes(req, res, next)`: Call categoryService.getAttributes(req.params.id), return { success: true, data }
- `createAttribute(req, res, next)`: Call categoryService.createAttribute(req.params.id, req.body), return 201 with { success: true, data }
- `updateAttribute(req, res, next)`: Call categoryService.updateAttribute(req.params.attributeId, req.body), return { success: true, data }
- `deleteAttribute(req, res, next)`: Call categoryService.deleteAttribute(req.params.attributeId), return { success: true, message: 'Attribute deleted' }

Export singleton: `export const categoryController = new CategoryController()`
