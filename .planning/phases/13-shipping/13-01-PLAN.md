    Create `apps/server/src/modules/shipping/shipping.validation.ts` with Zod schemas:

    1. `createShippingZoneSchema` -- validates: name (string 1-100), countries (array of 2-char ISO strings, min 1), states (optional array of strings, default []), freeShippingThreshold (positive int or null, optional), isActive (boolean, default true). Wrap in z.object({ body: ... }) for the validate middleware pattern.

    2. `updateShippingZoneSchema` -- same fields as create but all optional (partial).

    3. `createShippingMethodSchema` -- validates: zoneId (string cuid), name (string 1-100), description (string max 500, optional), rateType (enum FLAT_RATE | WEIGHT_BASED | PRICE_BASED), flatRate (positive int, nullable optional), weightRate (positive int, nullable optional), minWeight (positive float, nullable optional), maxWeight (positive float, nullable optional), priceThresholds (record of string to int, nullable optional), estimatedDaysMin (positive int, nullable optional), estimatedDaysMax (positive int, nullable optional), isActive (boolean, default true), position (int min 0, default 0). Add a `.refine()` that validates: FLAT_RATE requires flatRate, WEIGHT_BASED requires weightRate, PRICE_BASED requires priceThresholds.

    4. `updateShippingMethodSchema` -- partial update, same refine if rateType is provided.

    5. `calculateRateSchema` -- validates body: country (2-char string), state (optional string), cartSubtotal (positive int, cents), cartWeight (non-negative number in kg).

    Create `apps/server/src/modules/shipping/shipping.service.ts` with a `ShippingService` class:

    1. **Zone CRUD**: `createZone(data)`, `getAllZones(includeInactive?)`, `getZoneById(id)`, `updateZone(id, data)`, `deleteZone(id)`. Use `prisma.shippingZone` with `include: { methods: true }` on read operations.

    2. **Method CRUD**: `createMethod(data)`, `getMethodsByZone(zoneId)`, `updateMethod(id, data)`, `deleteMethod(id)`. Validate that zoneId exists before creating.

    3. **Zone matching**: `findMatchingZone({ country, state })` -- Two-tier priority:
       - Priority 1: Zone where countries has `country` AND states has `state` (state-specific match)
       - Priority 2: Zone where countries has `country` AND states is empty (country-wide match)
       - Return zone with active methods included, ordered by method position
       - Return null if no zone matches

    4. **Rate calculation**: `calculateShippingRate({ method, cartSubtotal, cartWeight })` -- returns rate in cents:
       - Check free shipping threshold FIRST: if `method.zone.freeShippingThreshold` exists and `cartSubtotal >= threshold`, return 0
       - FLAT_RATE: return `method.flatRate ?? 0`
       - WEIGHT_BASED: validate cartWeight against minWeight/maxWeight, return `Math.round(cartWeight * method.weightRate)`
       - PRICE_BASED: parse priceThresholds as `Record<string, number>`, sort keys descending, find first threshold where cartSubtotal >= threshold, return its value
       - Throw AppError(400) for weight limit violations or unknown rate types

    5. **Get available methods**: `getAvailableShippingMethods({ country, state, cartSubtotal, cartWeight })` -- calls findMatchingZone, then maps each method through calculateShippingRate, filtering out methods that throw (e.g., weight exceeded). Returns array of `{ id, name, description, rate, estimatedDays }`.

    Export singleton: `export const shippingService = new ShippingService();`

    Use `import { prisma } from '@repo/db';` and `import { AppError } from '../../common/middleware/error-handler.js';`. Follow the exact same class-based service pattern as product.service.ts.

    All monetary values are integers (cents). Never use floats for money.
