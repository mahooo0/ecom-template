2. Create apps/server/src/modules/search/search.service.ts:
   - Import MeiliSearch from 'meilisearch' and config
   - Create SearchService class with:
     - private client: MeiliSearch (initialized in constructor from config.meilisearchHost and config.meilisearchMasterKey)
     - private index (client.index('products'))
     - async initializeIndex(): Configure settings BEFORE adding documents:
       - searchableAttributes: ['name', 'brandName', 'categoryName', 'description', 'sku'] (ordered by importance)
       - filterableAttributes: ['categoryId', 'brandId', 'status', 'price', 'productType', 'categoryPath']
       - sortableAttributes: ['price', 'createdAt', 'updatedAt', 'name']
       - rankingRules: ['words', 'typo', 'proximity', 'attribute', 'sort', 'exactness']
       - faceting: { maxValuesPerFacet: 100, sortFacetValuesBy: { '*': 'count' } }
       - pagination: { maxTotalHits: 1000 }
       - typoTolerance: { enabled: true, minWordSizeForTypos: { oneTypo: 5, twoTypos: 9 } }
     - async search(query: string, options?: { limit?: number; offset?: number; filter?: string | string[]; facets?: string[]; sort?: string[] }): Calls this.index.search() with attributesToHighlight: ['name', 'description'] and returns results
     - async addDocuments(documents: SearchDocument[]): Calls this.index.addDocuments()
     - async deleteDocument(id: string): Calls this.index.deleteDocument()
     - async deleteAllDocuments(): Calls this.index.deleteAllDocuments()
     - async updateSynonyms(synonyms: Record<string, string[]>): Calls this.index.updateSynonyms()
     - async getSynonyms(): Calls this.index.getSynonyms()
     - async updateStopWords(stopWords: string[]): Calls this.index.updateStopWords()
     - async getStopWords(): Calls this.index.getStopWords()
     - async updateRankingRules(rules: string[]): Calls this.index.updateRankingRules()
     - async getRankingRules(): Calls this.index.getRankingRules()
     - async getSettings(): Calls this.index.getSettings()
   - Export singleton: export const searchService = new SearchService()

3. Create apps/server/src/modules/search/sync.service.ts:
   - Import prisma from '@repo/db', SearchDocument from './types.js', searchService from './search.service.js', eventBus from '../../common/events/event-bus.js'
   - Create SyncService class with:
     - buildSearchDocument(product: any): SearchDocument - denormalize product with brand/category into flat SearchDocument. Use product.category.name for categoryName, product.category.path for categoryPath, product.brand?.name for brandName. Convert dates to getTime() unix timestamps.
     - async indexProduct(productId: string): Fetch product from prisma with include: { category: true, brand: true }, build search document, call searchService.addDocuments([doc]). Fire-and-forget pattern (caller should .catch() errors). Skip if product status is not 'ACTIVE' (delete from index instead).
     - async deleteProduct(productId: string): Call searchService.deleteDocument(productId)
     - async fullSync(): Batch sync all ACTIVE products. Use cursor-based pagination with BATCH_SIZE = 10000. For each batch: fetch from prisma with include: { category: true, brand: true } and where: { status: 'ACTIVE' }, build search documents, call searchService.addDocuments(). Log progress.
     - registerEventListeners(): Register listeners on eventBus:
       - 'product.created' -> this.indexProduct(data.productId).catch(err => console.error('Search index failed:', err))
       - 'product.updated' -> this.indexProduct(data.productId).catch(err => console.error('Search index failed:', err))
       - 'product.deleted' -> this.deleteProduct(data.productId).catch(err => console.error('Search index failed:', err))
   - Export singleton: export const syncService = new SyncService()
   - At module level, call syncService.registerEventListeners() so listeners are registered on import

IMPORTANT: Do NOT modify apps/server/src/modules/product/product.service.ts - it already emits the events we need. The sync service just listens to them.
