      - Throws AppError(409, 'Insufficient stock') if available < qty
      - Throws AppError(404, 'Inventory item not found') if item doesn't exist
    - commitReservation(checkoutSessionId):
      - Find all RESERVATION movements with this reference
      - Inside transaction: for each, decrement both quantity and reserved on the inventoryItem, create SALE movement with same reference
      - Emit inventory.stockUpdated event
      - Check low stock threshold after commit
    - releaseReservation(checkoutSessionId):
      - Find all RESERVATION movements with this reference that don't have a matching RESERVATION_RELEASE
      - Inside transaction: decrement reserved on inventoryItem, create RESERVATION_RELEASE movement
      - Idempotent: if already released (no unreleased reservations found), return silently
    - releaseExpiredReservations():
      - Find RESERVATION movements older than 15 minutes with no matching RESERVATION_RELEASE (matched by reference)
      - Release each one
      - Return count of released reservations
  </behavior>

- reserveStock increments reserved within transaction mock
- reserveStock throws 409 when available < qty
- commitReservation creates SALE movement and decrements quantity + reserved

Then add the four methods to InventoryService in inventory.service.ts:

reserveStock: Use prisma.$transaction(async (tx) => { ... }) pattern. Inside transaction:
  1. tx.inventoryItem.findUnique({ where: { variantId_warehouseId: { variantId, warehouseId } } })
  2. Check available = item.quantity - item.reserved >= qty
  3. tx.inventoryItem.update({ where: { id: item.id }, data: { reserved: { increment: qty } } })
  4. tx.stockMovement.create({ data: { inventoryItemId: item.id, quantity: -qty, reason: 'RESERVATION', reference: checkoutSessionId } })

commitReservation:
  1. Find reservation movements: prisma.stockMovement.findMany({ where: { reason: 'RESERVATION', reference: checkoutSessionId } })
  2. prisma.$transaction for each: decrement quantity by abs(movement.quantity), decrement reserved by abs(movement.quantity), create SALE movement
  3. After commit: check threshold and emit events

releaseReservation:
  1. Find unreleased: prisma.stockMovement.findMany with reason RESERVATION and reference = checkoutSessionId, then check no RESERVATION_RELEASE exists for same reference+inventoryItemId
  2. For each: prisma.$transaction: decrement reserved, create RESERVATION_RELEASE movement
  3. If none found, return silently (idempotent)

releaseExpiredReservations:
  1. const cutoff = new Date(Date.now() - 15 * 60 * 1000)
  2. Find RESERVATION movements created before cutoff
  3. Filter out those that already have RESERVATION_RELEASE for same reference
  4. Group by reference and call releaseReservation for each unique reference
  5. Return count

IMPORTANT: All reservation operations MUST be inside $transaction. Never read-then-write outside transaction. Use { increment: N } / { decrement: N } not direct assignment.

```typescript
import cron from 'node-cron';
import { inventoryService } from './inventory.service.js';

export function startReservationCleanup(): void {
  // Run every minute to release expired reservations (15-min TTL)
  cron.schedule('* * * * *', async () => {
    try {
      const released = await inventoryService.releaseExpiredReservations();
      if (released > 0) {
        console.log(`[Inventory] Released ${released} expired reservation(s)`);
      }
    } catch (error) {
      console.error('[Inventory] Reservation cleanup error:', error);
    }
  });

  console.log('[Inventory] Reservation cleanup cron started (every 1 min, 15-min TTL)');
}
```
