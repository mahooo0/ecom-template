<task type="auto" tdd="true">
  <name>Task 1: Add filterProducts method with JSONB attribute queries and OR/AND logic</name>
  <files>apps/server/src/modules/product/product.service.ts, apps/server/src/modules/product/product.schemas.ts, apps/server/src/modules/product/product.service.test.ts</files>
  <behavior>
    - filterProducts returns products matching category path (startsWith on materialized path)
    - Price range filter: gte minPrice, lte maxPrice (values in cents)
    - Brand filter: brandId in [...brandIds] (OR within group)
    - Attribute filter: JSONB path queries with OR within same attribute key, AND across different keys
    - Availability: in_stock = variants.some.stock > 0, out_of_stock = variants.every.stock = 0
    - Returns { products, total, totalPages } with pagination
  </behavior>
  <action>
1. In product.schemas.ts, add a filterQuerySchema using Zod:
   - minPrice: z.coerce.number().optional()
   - maxPrice: z.coerce.number().optional()
   - brands: z.string().optional() (comma-separated brand IDs)
   - attributes: z.string().optional() (comma-separated "key:value" pairs)
   - availability: z.string().optional() (comma-separated: "in_stock", "out_of_stock")
   - categoryPath: z.string().optional()
   - page: z.coerce.number().default(1)
   - limit: z.coerce.number().default(20)
   - sortBy: z.string().default('createdAt')
   - sortOrder: z.enum(['asc', 'desc']).default('desc')

2. In product.service.ts, add filterProducts(filters) method:
   - Build Prisma where clause from filters
   - Category: where.category = { path: { startsWith: categoryPath } }
   - Price: where.price = { gte: minPrice, lte: maxPrice }
   - Brands: where.brandId = { in: brandIds } (OR logic)
   - Attributes: Parse "key:value" strings, group by key, build AND array where each group uses OR on JSONB path
     ```
     where.AND = Object.entries(grouped).map(([key, values]) => ({
       OR: values.map(value => ({ attributes: { path: [key], equals: value } }))
     }))
     ```
   - Availability: in_stock → variants: { some: { stock: { gt: 0 } } }, out_of_stock → variants: { every: { stock: { equals: 0 } } }
   - Always filter by status: 'ACTIVE'
   - Return paginated results with total count

3. Convert the it.todo() stubs in product.service.test.ts for FILT-01, FILT-05, FILT-07 into real tests that verify the filterProducts method builds correct Prisma where clauses against the mocked Prisma client.
  </action>
